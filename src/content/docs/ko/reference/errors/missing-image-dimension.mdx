> Missing width and height attributes for `IMAGE_URL`. When using remote images, both dimensions are required in order to avoid cumulative layout shift (CLS).

## 무엇이 잘못되었나요?

원격 이미지의 경우 `width` and `height`는 원본 파일에서 추론할 수 없습니다. 따라서 CLS를 방지하려면 이 두 속성이 항상 필요합니다.

원격 이미지의 경우 원본 파일에서 `width` 및 `height`를 자동으로 추론할 수 없습니다. CLS (누적 레이아웃 변경)를 방지하려면 이 두 속성을 지정하거나 [inferSize](/ko/reference/modules/astro-assets/#infersize)를 `true`로 설정하여 원격 이미지의 원래 크기를 가져와야 합니다.

대신, 이미지가 `src` 폴더 안에 있다면 이미지를 가져오려고 했을 것입니다. [자세한 내용은 가져오기 안내서](/ko/guides/imports/#기타-자산)를 참조하세요.
