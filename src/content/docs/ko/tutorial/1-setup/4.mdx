이 튜토리얼에서는 GitHub를 사용하여 저장소를 저장하고 웹 호스트에 연결합니다. 원하는 온라인 Git 제공업체를 사용해도 됩니다.

:::note
이미 git에 익숙하고 자신만의 워크플로가 있는 경우 선호하는 방법을 사용하여 프로젝트에 대한 새 GitHub 저장소를 생성하세요. 그리고 다음 페이지: ["사이트를 웹에 배포"](/ko/tutorial/1-setup/5/)로 건너뛰세요.
:::

## GitHub에 저장소 만들기

GitHub에 로컬 코드를 저장하는 방법에는 여러 가지가 있지만 이 튜토리얼에서는 명령줄에서 git을 사용할 필요가 없는 방법을 안내합니다.

<Steps>
1. 브라우저에서 GitHub.com에 로그인하고 화면 오른쪽 상단의 <kbd>+</kbd>를 클릭하여 새 저장소를 만듭니다.

2. 저장소 이름을 선택합니다. 프로젝트 폴더와 이름이 같을 필요는 없습니다.

3. 옵션이 표시되지만 기본값을 변경할 필요는 없습니다. 아래로 스크롤하여 <kbd>Create Repository</kbd> 버튼을 클릭하세요.

4. 다양한 설정 다음 단계가 표시되지만 그 중 어느 것도 사용할 필요는 없습니다. 저장소의 URL을 기록해 두세요. 이제 아무 작업도 하지 않고 이 페이지를 종료할 수 있습니다.
</Steps>

## 로컬 코드를 GitHub에 커밋

마지막 섹션에서는 페이지 콘텐츠를 변경했습니다. 이는 프로젝트 파일이 변경되었으며 VS Code가 "Source" 메뉴 아이콘 위에 숫자를 표시해야 함을 의미합니다. 이 소스 탭은 GitHub에서 파일을 업데이트하기 위해 정기적으로 이동하는 곳입니다.

<Steps>
1. VS Code에서 Source Control 탭을 클릭하여 변경된 파일 목록을 확인합니다. `git`을 설치해야 한다는 메시지가 표시되면 제공된 지침을 따른 다음 VS Code를 다시 로드하세요.

2. 커밋 메시지 위의 <kbd>•••</kbd> "점 3개" 메뉴를 클릭하고 <kbd>Remote</kbd> > <kbd>Add Remote</kbd>를 선택합니다.

3. <kbd>Add remote from GitHub</kbd>를 선택하세요. 필요한 경우 인증 단계를 따른 다음 VS Code로 돌아가 이 작업을 반복합니다.

4. GitHub의 모든 저장소 목록이 표시됩니다. 이 프로젝트를 위해 생성한 것을 선택하세요. 프로젝트가 표시되지 않으면 GitHub URL을 직접 붙여넣으세요. 이 저장소에 로컬 이름을 지정하라는 요청을 받을 수도 있습니다. 원하는 이름을 선택할 수 있습니다.

5. 메뉴 창 상단에는 **commit message** (파일 변경 사항에 대한 설명)를 입력할 수 있는 공간이 있습니다. `initial commit`을 입력하고 <kbd>Commit</kbd> 버튼을 눌러 변경 사항을 커밋하세요.

6. "staged" 커밋이 없다는 메시지와 함께 스테이징할지 묻는 메시지가 표시될 수 있습니다. <kbd>Always</kbd>을 클릭하고 계속하세요.

7. 마지막으로 변경된 파일 목록이 <kbd>Publish</kbd> 버튼으로 대체되어야 합니다. 커밋된 변경 사항을 GitHub에 보내려면 이 버튼을 클릭하세요.
</Steps>

### GitHub에서 프로젝트를 확인

프로젝트가 GitHub에 성공적으로 저장되었는지 확인하려면 GitHub.com을 방문하여 계정에서 저장소 목록을 확인하세요. 생성한 새 파일을 선택하고 Astro 프로젝트 파일이 포함되어 있는지 확인하세요.
